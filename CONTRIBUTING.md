Obrigado por contribuir!

Regras rápidas:

- Abra um issue antes de grandes mudanças para discutir o design.
- Crie branches por feature: `feature/nome-da-feature`.
- Use PRs para enviar mudanças; inclua descrição do que foi alterado e por quê.
- Mantenha o código legível e com tipos (TypeScript).

Rodando localmente:

1. Instale dependências:

   npm install

2. Compile em modo watch durante o desenvolvimento:

   npm run build

3. Para verificar a build única (quando for subir um PR):

   npm run build:prod

Formato de commits: prefira mensagens descritivas no estilo `feat:`, `fix:`, `chore:`.

Obrigado pela colaboração!
